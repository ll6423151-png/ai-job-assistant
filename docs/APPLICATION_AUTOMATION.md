# 智联招聘投递执行模块

## 安全边界

“即将投递”清单支持“批量准备预览（最多 5 条）”，用于集中读取登录状态、岗位页面和投递入口。该动作不调用任务 `submit`，不点击平台最终投递按钮。

每个任务仍必须独立完成岗位/公司/简历/沟通快照核对，并使用自己的确认令牌执行最终提交。系统不提供无人确认的批量真实投递；平台登录、验证码和风控由用户处理。

- 软件不保存招聘平台账号、密码、短信验证码或扫码凭证。
- 首次登录、二维码、验证码和安全校验必须由用户在 Edge 中完成。
- 每个岗位生成独立任务和不可变快照，禁止静默批量提交。
- 最终外部点击需要当前任务的确认框和一次性预览令牌。
- 平台未显示“已投递/已沟通/投递成功”时，任务标记为“需人工核验”，不会伪造成功记录。
- 控制投递频率；自动化仍可能触发平台风控或账号限制。
- 同一平台和岗位已有待处理、已提交或需核验任务时，禁止再创建重复任务。
- 默认所有支持平台合计每日最多进入 20 次外部提交，任意两次尝试至少间隔 60 秒；进入 `submitting` 后即计数，包括后续失败或需人工核验的尝试。

## 浏览器连接

本模块当前只启用智联招聘，通过本机 `web-access` CDP Proxy 复用 Edge 登录会话，默认地址为 `http://127.0.0.1:3456`。BOSS 直聘和实习僧适配器未注册、不会出现在 API 或前端选项中。

1. 在 Edge 打开 `edge://inspect/#remote-debugging`。
2. 勾选 `Allow remote debugging for this browser instance`。
3. 运行 web-access 连接检查。
4. 打开软件“投递执行”页面后会自动检查智联连接状态。若显示“未打开智联招聘页面”，可点击“打开登录”，也可直接点击“搜索并生成清单”，系统会自动打开所需页面。
5. 登录、验证码或安全校验仍需手动完成；完成后点击“检查状态”，或重新执行搜索。

## 投递状态机

`draft -> awaiting_login | awaiting_confirmation -> submitting -> submitted | verification_required | failed`

重复调用已完成任务的提交接口会返回冲突，不会创建第二条投递记录。

任务预览会保存当次 `daily_limit` 和 `cooldown_seconds` 策略快照。默认值可通过后端环境变量 `APPLICATION_AUTOMATION_DAILY_LIMIT` 和 `APPLICATION_AUTOMATION_COOLDOWN_SECONDS` 调整。当前保护使用数据库逻辑查询，适合本机单进程模式；多实例部署前仍需增加数据库锁或集中式限流，不能把当前实现视为严格的分布式并发配额。

## 使用流程

1. 在“用户中心”至少填写目标岗位、求职城市和最低期望月薪。
2. 在“简历管理”将一份简历设为主简历并标记为“可投递”。
3. 在“投递执行”点击“一键开始自动投递准备”。系统按用户中心的目标岗位和城市搜索智联官网，读取搜索卡片，并仅为接近硬规则的候选补充读取详情 JD。
4. 系统按薪资、黑名单、智联链接、城市、目标岗位/JD、学历、经验、职级和公司规模生成“即将投递”清单；也可继续手工录入真实岗位链接。官网明确显示 0 元或无薪时，岗位被拒绝且公司名称自动加入精确匹配黑名单；薪资面议或未知会被拒绝，但不会被误判为 0 元公司。
5. 对不准备投递的岗位勾选“不投递”；任务会进入 `cancelled`，不会继续打开平台页面。
6. 对单个岗位点击“预览并确认”，系统先打开岗位页并生成预览。
7. 核对页面标题、登录状态、岗位和简历，勾选当前任务确认框后点击“确认并投递”。若智联弹出简历选择框，系统按任务简历名称选择；若直接投递，则使用智联账号的默认简历。
8. 页面出现“已投递/投递成功”才记录成功，否则进入“需人工核验”。只有成功标识已确认后，系统才尝试打开沟通入口并填写任务保存的沟通内容；存在真实消息输入框和发送按钮时才点击发送，并要求输入框清空或页面出现已发送内容作为证据。
9. 当前智联 PC 官网的“立即沟通”只显示下载智联 APP 的提示，没有网页消息输入框。此时投递任务仍保留 `submitted`，沟通状态为 `manual_required`，页面提供原文复制；不得把该状态写成“已发送给 HR”。

“职位搜索”页允许输入本次临时关键词，例如“AI漫剧”。该关键词同时用于智联搜索和本次岗位相关性判断，但不会改写用户中心的长期目标岗位。搜索结果优先展示符合且已导入的候选，并区分“新导入”和“已存在并更新”。保守运行时每批限制为少量页面和少量导入，完成后停止等待用户确认；已有 JD 的本地岗位直接复用，不重复读取详情。

投递执行页的一键入口为 `POST /api/application-automation/auto-apply/run`：它先校验用户中心和可投递主简历，再在一次请求中执行智联搜索导入与本地草稿计划，避免前端串联两个请求产生“岗位已导入但清单未生成”的部分成功。独立官网搜索接口仍为 `POST /api/application-automation/zhaopin/search-import`，独立计划接口仍为 `POST /api/application-automation/auto-apply/plan`；二者供职位搜索页和恢复/诊断使用。所有计划接口只创建本地 `draft` 任务，不执行浏览器投递点击。取消接口为 `POST /api/application-automation/tasks/{task_id}/cancel`。已进入 `submitting`、`submitted` 或 `verification_required` 的任务不能通过该接口取消。
