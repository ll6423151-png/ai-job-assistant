## 1. 项目概述与目标

AI Job Assistant 是一个面向求职全流程的单体 Web 应用，帮助用户维护求职资料、管理职位、分析 JD 匹配度、生成并审核沟通内容、记录投递进度、开展 AI 模拟面试，并在明确安全边界内辅助招聘平台的单条投递。

当前产品目标：

- 复用同一套简历、职位、匹配和投递数据；账号体系只在现有 Next.js/FastAPI/SQLAlchemy 架构上增量实现，不重做整体 UI 或系统架构。
- 对简历优化、沟通定稿、投递记录和外部平台操作保留显式用户确认。
- 真实招聘平台登录、扫码、验证码和安全校验由用户手动完成。
- 默认使用可离线运行的确定性本地 AI；OpenAI 提供商保持可插拔。
- 自动投递候选必须同时满足公司黑名单、岗位来源和最低月薪规则。

不在当前能力范围内：无人确认的批量投递、绕过平台登录/验证码/风控、保存招聘平台账号密码、无人值守的跨平台职位搜索、把测试岗位当作真实投递对象。

## 2. 技术栈、系统架构和目录结构

### 2.1 当前技术栈

- 前端：Next.js 15、React 19、TypeScript、Tailwind CSS 3、Lucide React。
- Android：原生 Java WebView 壳、Android Gradle Plugin 8.9.2、Gradle 8.11.1、compile/target SDK 35、min SDK 24；不复制业务数据或重写现有前后端。
- 后端：FastAPI、Pydantic Settings、SQLAlchemy 2、Uvicorn。
- 认证：QQ 邮箱账号、Argon2id 密码哈希、数据库可撤销服务端会话、HttpOnly Cookie、QQ SMTP 验证码。
- 数据库：默认配置为 PostgreSQL 16；本机在线模式和开发数据库使用 SQLite；测试使用内存 SQLite。
- 辅助服务：Docker Compose 声明 Redis 7，但当前业务代码未发现 Redis 读写或任务队列实现。
- AI：`local` 确定性面试提供商；可选 OpenAI 文本、转写和 TTS；本地语音转写使用 faster-whisper，简繁转换使用 OpenCC。
- 文档解析：PDF、DOCX、TXT、Markdown；使用 pypdf 与 python-docx。
- 浏览器桥接：FastAPI 通过本机 `web-access` CDP Proxy（默认 `127.0.0.1:3456`）操作用户已登录的 Edge 页面。
- 本机在线模式：应用账号登录 + Next.js `/api` 反向代理 + Cloudflare Quick Tunnel。
- 测试：pytest、FastAPI TestClient；前端使用 Node 内置测试覆盖一键投递客户端、候选排序和实时语音通话状态逻辑，并保留 TypeScript 检查与 Next.js 构建命令。

### 2.2 系统架构

```text
浏览器
  -> Next.js 单页标签式工作台
       -> /api 重写到 FastAPI
            -> SQLAlchemy -> SQLite（本机）/ PostgreSQL（Compose）
            -> 本地 AI / 可选 OpenAI
            -> 本地 Whisper
            -> BrowserBridge -> web-access CDP Proxy -> Edge 招聘平台会话

Android APK
  -> 原生 WebView（文件选择、麦克风权限、Cookie、外链）
       -> 公网 HTTPS Next.js -> 同一套 FastAPI 与用户数据库

公网访问（当前本机模式）
  -> Cloudflare Quick Tunnel
       -> Next.js 登录路由与会话 Cookie
             -> 前端与 /api 代理
```

前端以 `frontend/src/app/page.tsx` 维护登录后的 12 个业务标签，并新增欢迎、登录、注册和找回密码路由。后端 `backend/app/main.py` 统一注册 `/api` 路由；启动时创建缺失表、初始化本机管理员，并对旧 SQLite/PostgreSQL 表执行可重复的 `user_id` 兼容升级与历史数据回填。

## 3. 数据库结构及主要模型

启动方式仍以 SQLAlchemy `create_all` 为基础，当前没有正式 Alembic 版本链。账号节点新增了启动兼容迁移：在已备份数据库上增加 `user_id`、回填历史归属、建立索引，并把公司黑名单全局唯一约束升级为“用户 + 公司”唯一。所有业务根模型通过 `TenantOwned` 和统一会话查询条件自动限定当前用户；面试轮次/报告通过所属面试会话间接隔离。其他业务关联字段仍主要依赖 API 逻辑校验，主要外键补齐仍待 Alembic 节点。

| 表 | 主要用途与关键字段 |
|---|---|
| `users` | 登录账号；用户名、QQ 邮箱、Argon2id 密码哈希、启用/管理员/邮箱验证/历史数据所有者标记。 |
| `auth_sessions` | 服务端会话；只保存令牌 SHA-256 哈希、有效期、最后活动、撤销时间及有限客户端信息。 |
| `email_verification_codes` | 注册/登录/重置密码验证码；只保存 HMAC 哈希、用途、5 分钟有效期、失败次数和消费时间。 |
| `user_profiles` | 每用户一份资料与偏好；目标岗位/城市/公司规模为 JSON，包含 `salary_min`、`salary_max`。 |
| `resumes` | 简历版本；标题、目标岗位、版本号、草稿/可投递状态、正文、备注、主版本标记。 |
| `jobs` | 职位库；公司、城市、薪资上下限、学历、经验、规模、JD、来源平台、原始链接、状态、每用户隔离的收藏标记和本地归档标记。 |
| `job_search_exclusions` | 每用户、每组搜索条件、每个岗位链接的历史跳过记录；保存搜索签名、岗位快照、拒绝原因及最后出现时间，不等同公司黑名单。 |
| `blacklist_companies` | 每用户公司黑名单及内部岗位屏蔽词；`user_id + company_name` 唯一，支持 `exact`/`contains`、原因和启用状态。 |
| `match_analyses` | JD 匹配结果；分数、命中/缺失关键词、原因和建议快照。 |
| `resume_optimizations` | 简历优化草稿；原文、建议稿、建议、警告及 `draft/applied/rejected` 状态。 |
| `greeting_messages` | 沟通内容；职位/简历快照、语气、正文及 `draft/approved` 状态。 |
| `application_records` | 本地投递记录；渠道、状态、确认标记、投递/跟进时间、备注和 JSON 状态时间线。 |
| `automated_application_tasks` | 平台投递任务；不可变职位/简历/沟通快照、预览、外部结果、一次性确认令牌和状态。 |
| `resume_assets` | 上传文件元数据与解析结果；不保存原始文件，只关联解析后创建的简历。 |
| `interview_sessions` | 面试配置、引用与不可变快照、题数、状态、提供商和时间。 |
| `interview_turns` | 问答轮次；`session_id + request_id` 唯一约束用于答案幂等。 |
| `interview_reports` | 每会话唯一报告；总分、维度分、摘要、优势、改进、证据和行动建议。 |

本机 `backend/dev.db` 在 2026-07-16 账号迁移后为 16 张表。迁移前备份为 `backend/backups/dev-pre-auth-20260716-023245.db`，长度与原库同为 421888 字节。副本迁移连续执行两次成功；原库历史数据归属管理员 `admin`：4 份简历、13 个岗位、3 条投递记录、10 场面试。数据量只描述本机状态，不是产品能力证明。

## 4. API 接口和主要页面

### 4.1 API

OpenAPI 由 FastAPI 自动生成，主要接口如下：

- 公开基础：`GET /api/health`。
- 账号认证：`POST /api/auth/email-codes|register|login|code-login|reset-password|logout`、`GET /api/auth/me`。验证码接收邮箱必须为 QQ 邮箱；本机管理员同时支持用户名登录。
- 登录后基础：`GET /api/modules`；除健康检查和认证接口外，所有业务接口均要求有效会话。
- 用户中心：`GET/POST/PUT/DELETE /api/user-profile`。
- 简历：`GET/POST /api/resumes`、`GET/PUT/DELETE /api/resumes/{id}`、`POST /api/resume-uploads`。
- 职位：`GET/POST /api/jobs`、`GET/PUT/DELETE /api/jobs/{id}`、`POST /api/jobs/{id}/favorite`、`POST /api/jobs/bulk-delete`、`POST /api/jobs/bulk-restore`；列表支持 `favorite_only=true`，批量清除先预览准确范围再把岗位归档，撤销上次清除可恢复归档岗位。
- 黑名单：`GET/POST /api/blacklist`、`GET /api/blacklist/check`、`GET/PUT/DELETE /api/blacklist/{id}`。
- JD 匹配：`GET/POST /api/matches`、`GET/DELETE /api/matches/{id}`。
- 简历优化：`GET/POST /api/optimizations`、`GET/PUT/DELETE /api/optimizations/{id}`、`POST .../apply`、`POST .../reject`。
- 沟通内容：`GET/POST /api/greetings`、`GET/PUT/DELETE /api/greetings/{id}`、`POST .../approve`。
- 投递记录：`GET/POST /api/applications`、`GET/PUT/DELETE /api/applications/{id}`、`POST .../confirm`、`POST .../status`。
- 数据统计：`GET /api/analytics/overview`。
- 平台适配器：`GET /api/platform-adapters`、`POST .../{key}/preview`、`POST .../{key}/import`。
- 面试：`GET/POST /api/interviews`、`GET /api/interviews/{id}`、`POST .../start|answers|complete|cancel`、`GET .../report`。
- 面试音频：`POST /api/interview-audio/transcriptions`、`POST /api/interview-audio/speech`。
- 投递执行：平台列表、打开登录、登录状态、智联官网搜索导入、单请求一键搜索/计划、独立自动计划、任务取消、任务列表/创建、任务准备和任务提交，前缀为 `/api/application-automation`；投递执行页一键接口为 `POST /api/application-automation/auto-apply/run`，独立官网搜索接口为 `POST /api/application-automation/zhaopin/search-import`，当前条件历史跳过记录清除接口为 `POST /api/application-automation/zhaopin/search-exclusions/clear`。

### 4.2 主要页面

公开页面包括 `/welcome`、`/login`、`/register` 和 `/forgot-password`，支持密码显隐、两种登录模式、验证码倒计时、字段校验及错误反馈。未登录访问主页会跳转欢迎页；登录过期时主页会回到登录页。登录后 `frontend/src/app/page.tsx` 默认打开“AI 模拟面试”，共 12 个标签，页头显示当前账号和退出按钮：

1. AI 模拟面试：选择/跳过/上传简历，选择或手填岗位，配置题型/压力/题数，连续作答、录音转写、朗读、报告、历史和运营视图。
2. 投递执行：当前仅提供智联招聘连接、登录检查、官网搜索导入、候选岗位过滤、0/无薪公司拉黑、沟通内容快照、任务预览、逐条确认、入口点击、成功核验和投递后沟通状态。
3. 平台适配器：查看注册能力，预览并确认导入结构化职位；当前界面明确不抓取平台。
4. 数据统计：投递指标、状态/渠道/匹配分布、14 天趋势和最近记录。
5. 投递记录：创建本地待确认记录、确认已投递、状态流转、跟进和时间线。
6. 沟通内容：按职位/简历/语气生成、编辑、确认定稿和复制。
7. 简历优化：基于匹配记录生成草稿，编辑后确认应用或拒绝。
8. JD 匹配：选择职位与简历，生成并查看可解释评分。
9. 公司黑名单：规则 CRUD 与公司名称检查。
10. 职位搜索：职位 CRUD、多条件筛选、黑名单结果控制、直接收藏/取消收藏和仅看收藏。
11. 简历管理：版本 CRUD、主版本、正文和备注。
12. 用户中心：个人信息、求职方向、薪资和公司规模偏好。

### 5.5 限制与风险

- Quick Tunnel URL 可变且无可用性保证；本机休眠/关机、Edge 退出、远程调试未授权、浏览器桥接或本机代理不可用都会中断相关能力。
- 平台 DOM、按钮文本和成功标识随时可能变化；未知状态必须进入人工核验，不能写成成功。
- FastAPI 业务接口本身已执行会话认证和用户隔离；直接暴露后端端口不会绕过业务鉴权，但仍应由防火墙限制，避免扩大攻击面。
- `create_all` 只能创建缺失表；本节点兼容迁移只覆盖账号所需表、`user_id`、索引和两个唯一约束，后续字段演进仍应建立正式 Alembic 版本链。
- Docker 后端使用 `--reload`、前端使用 `npm run dev`，属于开发启动方式；Redis 目前只是保留配置。
- 生产构建本轮未重跑，因为现有 `next start` 正在使用同一 `.next` 目录，重建可能干扰在线实例；历史节点报告记录构建通过。
- pytest 有 Starlette TestClient/httpx 弃用警告，需要后续依赖兼容处理。
- 智联 PC 的“立即沟通”当前只是 APP 下载引导，不提供网页消息输入框；该平台限制不能通过伪造 DOM 或绕过官方流程解决。
- Android WebView 只是现有产品客户端；智联搜索/投递仍依赖后端主机的 Edge + CDP 会话，手机上的智联 App 登录不会自动同步给后端浏览器会话。

## 7. 项目记忆

### 7.1 关键决策

- 保持 Next.js + FastAPI 单体应用和登录后的单页标签式 UI，不新建独立项目；账号系统以增量路由、模型和统一隔离层接入，不重做业务架构。
- Android 采用现有前端的原生 WebView 客户端，不复制 API 或数据库；正式包只接受构建时注入的公网 HTTPS 地址，外部招聘平台链接由系统浏览器处理。
- QQ 邮箱由注册/登录用户输入并作为验证码收件地址；系统使用一个后端配置的 QQ SMTP 发件箱，不保存普通用户 QQ 密码或授权码。
- 历史数据不自动交给第一个注册用户；迁移给明确的本机管理员 `admin`，新 QQ 账号得到空白工作区，后续转移必须经过管理员确认。
- 外部平台操作是“人工登录 + 单条预览 + 单条确认 + 自动点击 + 结果核验”，不是无人值守批量投递。
- 任何不明确的外部结果进入 `verification_required`，不能伪造 `submitted`。
- 沟通内容只有在投递成功标识已验证后才能尝试发送；沟通发送失败不得抹掉已经验证的投递成功，必须独立记录 `sent`、`manual_required` 或 `verification_required`。
- 仅对官网明确显示 0 元/无薪的公司自动建立精确黑名单；薪资未知或面议只拒绝岗位，不推断公司必然无薪。
- 简历优化不得自动补写不存在的经历；面试评分必须返回回答证据。
- 本地 AI 是默认可用路径；OpenAI 失败时面试文本生成回退本地。
- 实时语音面试优先复用浏览器语音能力和现有 HTTP 面试状态机；在未配置并验证实时音频提供商前，不把自然轮流对话描述为模型级全双工。
- 上传原文件只在请求内解析，数据库保存元数据和提取文本，不持久化原文件。

### 7.2 环境配置

- PostgreSQL/Compose：`DATABASE_URL` 默认 `postgresql+psycopg://...@postgres:5432/jobai`。
- 本机在线：启动脚本显式使用 `sqlite:///./dev.db`、`AI_PROVIDER=local`、后端 `127.0.0.1:8000`、前端 `127.0.0.1:3000`。
- 浏览器桥接默认：`http://127.0.0.1:3456`；部署文档还提示本机出站代理 `127.0.0.1:7990` 的依赖。
- 应用认证：`AUTH_SECRET_KEY`、管理员强密码和 QQ SMTP 授权码仅从已忽略的本机环境文件读取；不得写入文档、日志、代码、报告或回复。公网 Cookie 必须启用 `Secure`。
- OpenAI：`.env.example` 中 Key 为空；是否存在机器级环境变量不等于项目已配置或已验证。
- Android 构建：`CAREERPILOT_BASE_URL` 在构建时写入 APK；release 拒绝 localhost、模拟器回环、非 HTTPS 和 `.invalid` 占位地址。隔离工具链位于已忽略的 `tools/android-build/`，签名材料位于已忽略的 `android/keystore/`，必须离线备份。

### 7.3 不可违反的安全边界

- “敏思跃动”保持启用的公司黑名单规则，除非用户明确要求变更。
- 自动投递候选的月薪下限不得低于 3000 元；无薪、面议且不能确认、缺失下限或低于门槛必须拒绝。
- 不保存平台账号、密码、验证码或扫码凭据。
- 登录、扫码、验证码和安全校验由用户手动完成。
- 最终平台提交必须由用户对当前单条任务显式确认；禁止静默批量操作。
- 测试只能使用假平台桥接、fixture 或明确测试数据；不得把测试岗位写成真实投递成功。
- 未经用户单次明确授权，不执行真实岗位最终提交、重新部署、停止现有在线服务、修改 Cloudflare 账号或系统启动项。

### 7.5 常见问题

- **为什么能搜索官网却仍不算无人值守全自动？** 搜索和导入可以自动完成，但平台登录/验证仍由用户处理，每个岗位必须先预览并逐条确认，系统不执行静默批量提交。
- **为什么注册验证码暂时不能收到？** 当前代码已接入 QQ SMTP，但本机尚未填写系统发件邮箱和 SMTP 授权码；按 `docs/AUTHENTICATION.md` 配置后才能真实发送。普通注册用户只填写收件 QQ 邮箱，不提供授权码。
- **默认管理员在哪里？** 本机数据库 `users` 表中存在 `admin`，历史业务数据归属该账号；首次本机测试密码为 `admin123`，只以 Argon2id 哈希存储。公网启动脚本拒绝该默认密码。
- **为什么“批量准备预览”后仍要逐条确认？** 批量按钮只减少打开岗位页面和读取入口的重复操作，不会点击平台最终投递按钮。每个任务的简历、岗位、沟通快照、确认令牌和成功标识仍独立核对，避免误选岗位导致批量真实投递。
- **当前为什么只有智联招聘？** 用户在 2026-07-15 明确要求先只使用智联招聘；BOSS 和实习僧已从注册表、API 和前端停用。
- **为什么显示“需人工核验”？** 页面点击后没有匹配明确成功文字，属于安全设计，不应强行标成功。
- **为什么软件里能填写沟通内容，却仍显示 APP 手动发送？** 软件会把内容保存到每条任务，并在投递成功后寻找官网消息输入框；当前智联 PC 只有 APP 下载入口，所以只能复制后在 APP 发送，系统不会伪造已发送状态。
- **为什么显示“未打开智联招聘页面”？** 这不代表账号退出或桥接损坏。点击“打开登录”或直接“搜索并生成清单”即可建立页面；登录/验证码仍由用户手动完成。
- **为什么本地可用但公网可能失效？** Quick Tunnel、本机电源、Edge、远程调试和代理都属于运行依赖。
- **为什么不能直接改数据库字段？** 当前无迁移工具；字段演进应先建立 Alembic 基线和备份。
- **为什么本轮不重跑 Next.js build？** 在线 `next start` 使用同一 `.next`，原地构建可能干扰当前服务；应在停止服务或隔离构建目录后验证。
- **实时语音面试是真正的全双工模型通话吗？** 当前是浏览器持续识别、AI 朗读、插话取消朗读和停顿自动提交形成的电话式轮流对话；没有配置 OpenAI Realtime，因此不宣称模型级同时双向音频流。最新版 Edge/Chrome 和麦克风权限是实时模式的必要条件，其他环境可继续使用文字或录音。
- **可爱音色为什么在不同电脑上不完全一样？** 播放由浏览器 `speechSynthesis` 完成，系统提供的中文语音包不同。软件会优先选择可识别的中文女声；找不到时使用 pitch/rate 回退。可在面试工作台把“音色”切换为“标准”。
- **为什么仍可能提示麦克风不可用？** 若浏览器同时不支持 `SpeechRecognition`、`MediaRecorder` 或 `getUserMedia`，无法建立实时通话；请改用最新版 Edge/Chrome、允许站点麦克风权限，或使用文字/手动录音。浏览器有录音能力但没有 `SpeechRecognition` 时，系统会自动走本地 Whisper 降级路径。
- **为什么 OpenAI 代码存在但仍算未验证？** 提供商代码、项目配置和真实 API 调用是三件事；本机在线模式当前明确强制 local。
- **为什么 APK 现在能安装却不能当作长期公开版？** 当前 release 连接的是本机 Quick Tunnel，地址临时且依赖本机服务常开。固定域名、稳定服务器、强密钥、SMTP 和备份就绪后，应使用同一签名重新构建；否则地址失效后 APK 无法登录。
- **为什么手机登录智联后，自动投递仍可能提示后端未登录？** 平台自动化运行在后端主机的 Edge/CDP 会话，不共享手机智联 App Cookie。现阶段仍需在后端所用浏览器人工登录，并保持逐条确认。

## 8. MVP 上线优先级冻结（2026-07-16）

本项目从本节点起停止新增非上线功能，按以下顺序推进，完成一项再进入下一项：

1. **优先级一：真实智联投递流程**。在不绕过用户确认、不保存平台密码、不探测性点击真实岗位的前提下，完成小批量真实岗位的搜索、导入、任务预览、简历选择分支、最终单条确认、成功标识和投递记录核对。智联 PC 没有网页沟通输入框时，保持 `manual_required`，不伪造自动发送成功。
2. **优先级一：Android 真机验收**。使用固定服务地址前的临时包完成安装、冷启动、登录保持、岗位/简历页面、文件上传、麦克风权限、深色模式和断网重试验收；固定域名确定后重打 release 包并再次验收。
3. **优先级一：QQ SMTP 实际可用**。由人工在私密环境配置系统 QQ 发件邮箱和 SMTP 授权码，完成注册验证码、验证码登录和找回密码的真实收件验收；不要求普通用户提供 QQ 密码或授权码。
4. **优先级一：固定域名正式部署**。配置固定 HTTPS 域名、持久化服务、`AUTH_COOKIE_SECURE=true`、强认证密钥和非默认管理员密码，完成健康检查、登录和业务 API 回归；临时 Quick Tunnel 只保留为过渡验证入口。
5. **优先级二：上线稳定性补强**。仅在以上四项跑通后处理正式数据库脚本/Alembic 基线、登录体验优化和系统健康检查页面。
