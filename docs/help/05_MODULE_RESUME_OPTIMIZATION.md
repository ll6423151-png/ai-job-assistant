# 模块 3：AI 简历优化

**当前状态：可用，本地确定性回退可运行。**

## 功能介绍

根据一份已保存的 JD 匹配记录生成可编辑优化草稿，突出已有证据、调整表达并提示技能缺口。

## 操作方法

1. 先在“JD 匹配”生成分析。
2. 打开“简历优化”，选择匹配记录。
3. 生成优化草稿并逐段核对。
4. 删除任何不真实或无法证明的内容。
5. 明确确认后应用到源简历；建议先保留原版本。

## 使用场景

针对岗位关键词调整表达；提高简历与 JD 的可读匹配度；准备不同岗位版本。

## AI 工作流

简历文本 + JD 差距 + 匹配证据 → 优化草稿 → 用户编辑 → 明确确认 → 更新简历。

## 用户输入与输出

- 输入：匹配记录、源简历、人工修改。
- 输出：优化草稿、警告和确认后更新的简历文本。

## 注意事项

缺失技能只作为警告，不会自动写成已掌握；AI 输出必须人工校验；不要覆盖唯一原稿。

## 常见问题

- **为什么建议没有直接写入？** 防止虚构经历。
- **如何撤回？** 使用保存的旧版本手工恢复。
- **OpenAI 不可用怎么办？** 系统使用本地回退，但语言多样性可能较低。
