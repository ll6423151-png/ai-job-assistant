# 完整使用流程：从注册到拿到 Offer

## 阶段 1：注册账号

1. 打开欢迎页，进入注册。
2. 输入本人可用的 QQ 邮箱并发送验证码。
3. 60 秒内不要重复发送；验证码 5 分钟有效。
4. 设置至少 8 位且包含字母和数字的密码。
5. 登录后确认顶部显示的是自己的账号。不同账号的数据相互隔离。

**完成标准：** 可以进入工作台，刷新或重开后仍保持登录。

## 阶段 2：填写用户中心

1. 填写真实姓名、当前城市和联系邮箱/手机号。
2. 填写目标岗位，可用逗号分隔多个方向。
3. 填写意向城市、学历和经验阶段。
4. 设置期望薪资；自动投递最低仍为 3000 元/月。
5. 公司规模是偏好，不应取代岗位方向、薪资和真实性判断。
6. 添加不希望看到的岗位关键词，例如“主播”。

**完成标准：** 目标岗位、城市和最低薪资存在；页面保存成功。

## 阶段 3：建立主简历

1. 上传 PDF、DOCX、TXT 或 Markdown。
2. 检查解析文本是否完整；扫描 PDF 先做 OCR。
3. 修正标题、目标岗位、正文和备注。
4. 将状态设为可投递，并设为主简历。
5. 保留一份未经优化的原始版本。

**完成标准：** 至少一份主简历且状态为可投递。

## 阶段 4：收集代表性职位

1. 优先使用已有职位和缓存，避免重复访问智联。
2. 手机端点击“搜索并导入”时，后端复用电脑 Edge/web-access；一次只搜 1 页、最多导入 5 条。
3. 也可以在智联 App 搜索后复制真实岗位链接，回软件新增岗位。
4. 点击“刷新已导入岗位”核对新增数量。
5. 达到 20–30 个代表性 JD 后停止抓取，转向分析关键词。

**完成标准：** 软件职位列表出现真实职位、公司、薪资、链接和尽可能完整的 JD。

## 阶段 5：执行硬性筛选

1. 排除公司黑名单和岗位屏蔽关键词。
2. 排除月薪低于 3000、无薪、零薪、薪资面议且无法确认达到 3000 的岗位。
3. 排除测试岗位、失效链接和明显不匹配城市/职级的岗位。
4. 公司“敏思跃动”不得进入自动投递候选。

**完成标准：** 候选岗位符合安全与薪资底线。

## 阶段 6：JD 匹配

1. 为重点岗位选择对应简历并生成匹配。
2. 阅读已匹配关键词、缺失关键词和理由。
3. 不要只看总分；核对职责、经验、学历和地点。
4. 将高价值缺口整理为简历和面试准备项。

**完成标准：** 每个重点岗位都有可解释匹配证据。

## 阶段 7：优化简历

1. 从匹配记录生成优化草稿。
2. 逐句确认内容真实且可证明。
3. 不把缺失技能写成已有经验。
4. 保存为岗位定制版本，确认后应用。

**完成标准：** 简历表达贴近 JD，但没有虚构经历。

## 阶段 8：准备沟通内容

1. 选择职位、简历和语气生成草稿。
2. 编辑为简短、具体、可验证的内容。
3. 审核定稿后关联到投递任务。
4. 智联 PC 无消息框时复制到 App 手动发送。

**完成标准：** 每个重点岗位有已审核沟通内容。

## 阶段 9：模拟面试

1. 选择岗位和对应简历。
2. 从标准难度开始，再使用挑战或高压追问。
3. 用实时语音回答；不可用时使用录音上传或文本。
4. 查看报告中的证据、结构和薄弱项。
5. 重做关键问题，直到答案稳定。

**完成标准：** 能清楚完成自我介绍、岗位动机、经历证据和反问。

## 阶段 10：生成投递清单

1. 在“投递执行”搜索并生成候选清单。
2. 查看每个岗位的薪资、公司、匹配理由和阻断项。
3. 对不投递岗位勾选排除。
4. 批量准备最多 5 条预览，但不要批量最终提交。

**完成标准：** 清单只保留用户愿意投递的真实岗位。

## 阶段 11：逐条确认投递

1. 打开单条预览，核对岗位链接、公司、简历和沟通快照。
2. 手动完成智联登录、扫码、验证码或安全验证。
3. 勾选当次确认后执行单条提交。
4. 只有页面出现“已投递/投递成功”才记为成功；否则人工核验。
5. 沟通无法自动发送时在智联 App 手动发送。

**完成标准：** 每条投递都有成功标识或明确人工核验状态。

## 阶段 12：跟进与面试

1. 在投递记录中维护渠道、状态、备注和跟进时间。
2. 收到回复后更新为筛选中或面试。
3. 面试前再次运行岗位模拟面试。
4. 面试后记录关键问题、反馈和下一步。

**完成标准：** 所有活跃岗位都有下一步和负责人（用户本人）。

## 阶段 13：复盘策略

1. 每周查看投递量、回复率、面试率和 Offer 率。
2. 回复率低：优先检查岗位筛选、简历匹配和沟通开场。
3. 面试率低：检查简历证据和岗位要求。
4. 面试后通过率低：集中训练报告中的薄弱维度。
5. 样本不足时不要过度解释比例。

## 阶段 14：记录 Offer

1. 收到正式 Offer 后，将记录状态更新为 Offer。
2. 在备注中记录岗位、薪资结构、地点、入职日期和需确认事项；不要保存身份证或银行卡信息。
3. 对比多个 Offer 时结合发展、岗位内容、合同和风险，不只看 AI 建议。
4. 完成选择后保留必要记录并清理不再需要的敏感材料。

**最终完成标准：** Offer 有真实书面依据，记录状态正确，用户完成独立判断和后续入职确认。
