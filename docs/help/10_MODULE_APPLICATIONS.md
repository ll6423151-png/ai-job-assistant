# 模块 8：投递记录管理

**当前状态：可用；本地记录与外部自动化任务是两条明确流程。**

## 功能介绍

创建待确认记录，关联职位、简历和已审核沟通内容，维护投递、筛选、面试、Offer、未通过和撤回状态及跟进时间线。

## 操作方法

1. 选择职位、简历、沟通内容和渠道。
2. 填写备注及下次跟进时间，创建待确认记录。
3. 自行完成投递后勾选确认，将本地状态更新为已投递。
4. 后续按招聘进度更新筛选、面试、Offer 等状态。
5. 查看时间线并在统计中复盘。

## 使用场景

统一管理手动投递和智联辅助投递；提醒跟进；记录面试和 Offer。

## AI 工作流

本模块以状态机和本地记录为主，不自动判断录用结果。外部自动化在单独的“投递执行”中，成功后创建/更新记录。

## 用户输入与输出

- 输入：职位、简历、渠道、备注、跟进时间、状态说明。
- 输出：投递记录、当前状态、状态历史和统计数据。

## 注意事项

“确认已投递”只更新本地记录，不操作平台；真实平台最终提交必须逐条确认；重复任务有幂等保护。

## 常见问题

- **确认按钮会真的投递吗？** 投递记录页不会，它只更新本地记录。
- **删除记录会撤回平台简历吗？** 不会，只删除本地记录。
- **Offer 如何记录？** 将已投递/面试状态推进到 Offer。
